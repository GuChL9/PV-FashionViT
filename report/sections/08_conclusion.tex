\section{总结、局限与后续工作}

\subsection{对研究问题的回答}

\textbf{第一，位置变化会不会明显影响分类？}会，而且影响远大于中心测试给人的印象。中心训练的 MLP、CNN 和 ViT-AbsPos 在 Center 上都有 $86\%$ 以上准确率，CNN 达到 $90.93\%$；换成 Large Shift 后，三者只剩 $14.22\%$、$13.80\%$ 和 $20.03\%$。这说明原任务中的高准确率很大一部分建立在“衣物总在画布中央”这一稳定条件上。一旦位置变化并伴随边缘裁剪，模型原有的分类能力不能直接迁移过去。

\textbf{第二，使用绝对位置编码的 Tiny ViT 能否自然适应位移？}不能。ViT-AbsPos 在 Large Shift 和位置网格上比中心训练的 MLP、CNN 略好，但 Grid Accuracy 仍只有约 $21\%$，Robust Drop 达到 66.35 个百分点。自注意力让远距离 patch 可以交换信息，却没有替代合适的训练分布；可学习绝对位置编码也仍会把 token 与固定画布坐标联系起来。

\textbf{第三，数据增强、Mean Pooling 和卷积 stem 是否有效？}三者的作用大小不同。加入平移、旋转和随机擦除后，ViT 的 Grid Accuracy 从约 $21\%$ 提升到约 $65\%$，这是最主要的变化；Mean Pooling 又带来约 1 个百分点的小幅改善；加入卷积 stem 后，HybridConv-ViT 的 Center、Large Shift、Rotation、Shift+Rotation 和 Grid Accuracy 都达到约 $81\%$。它没有取得最高的中心准确率，但位置改变、角度改变或二者同时出现时，成绩几乎不再下降。

因此，可以认为 HybridConv-ViT 已经达到本项目对“位置可变 FashionMNIST 分类器”的初步设计要求：它在保留较好分类准确率的同时，明显减弱了模型对中心位置和固定朝向的依赖，并且这一结果在五个随机种子下保持一致。它还不是一个完全解决问题的模型，例如 Shirt 等相似上装仍较难分类，极端位置的前景裁剪也没有消失；但相较于三个中心训练基线，它已经从“中心图像分类器”推进成了能够实际处理位置与角度变化的分类器。

\subsection{现有局限}

\begin{enumerate}
  \item \textbf{增强没有逐项拆开。}E4 同时加入平移、旋转和随机擦除；E6 又建立在增强与 Mean Pooling 之上。当前报告能确认组合方案有效，但还不知道每个组件单独贡献多少。
  \item \textbf{边缘裁剪仍构成混杂因素。}第 7 节已把无裁剪内层与可能裁剪外圈分开汇总，量化了边缘区域的额外下降；但外圈也比内层离中心更远，所以该对照仍不能把位置偏置和可见信息损失作纯因果分解。
  \item \textbf{早停依据偏向中心。}验证集使用 Center 协议，可能优先选择中心准确率较好的检查点，而不是网格或复合扰动最稳定的检查点。
  \item \textbf{训练预算有限。}CPU 配置采用 7 像素 patch、64 维 embedding、2 层 Encoder 和最多 12 个 epoch，足够完成课程项目的比较，但没有探索更大 ViT 或更长训练的上限。
  \item \textbf{Attention 解释有限。}rollout 样本按预测差异筛选，不能代表完整测试分布；注意力高响应也不等同于因果特征重要性。
  \item \textbf{效率证据不完整。}当前只可靠记录参数量和单次训练的 epoch time，尚未在统一硬件与 batch size 下重复测量推理延迟、峰值内存和吞吐量，也未统计 FLOPs。因此报告可以比较模型规模，但不宣称 Hybrid 已取得最佳计算效率。
\end{enumerate}

\subsection{后续可开展实验}

如果继续推进，优先补充几个能把现有现象讲得更清楚的实验：依次比较 Center、只平移、平移+旋转、平移+旋转+擦除；在相同增强下单独切换 CLS 与 Mean Pooling；再比较有无卷积 stem。这样可以更直接地判断每一步为何有效。位置编码也可以补充 Learnable、Sin-Cos 和 No-Pos 三种设置。

对于数据协议，后续可以设计可见像素比例受控的配对样本：在保持位移距离相近的情况下分别保留完整前景和模拟裁剪，从而比当前内外圈拆分更严格地区分纯位置变化与信息缺失。验证阶段也可同时监控 Center Accuracy 与扰动验证准确率，或者使用二者加权指标选择检查点。完成这些对照后，再考虑 GPU 配置、更小 patch 和更长训练。
